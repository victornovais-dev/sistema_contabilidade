# Project Map

## Summary

- Main project: Spring Boot 4.0.3 backend with Java 25.
- Build: Maven Wrapper with Spotless, Checkstyle, SpotBugs, PMD, Error Prone, JaCoCo and ArchUnit.
- Main UI: Thymeleaf templates plus static JS/CSS fallback assets.
- Public auth pages `/login` and `/primeiro_acesso` are served from `src/main/resources/static`.
- Authenticated pages are still served primarily through backend server-side rendering.
- `frontend-angular/` exists, but it is not the main runtime flow.
- Production auth is provider-based: `local|cognito`, with `prod` defaulting to `cognito`.

## Main Modules

- `auth`: login, refresh, logout, opaque session, cookies, authenticated user routes.
- `usuario`: CRUD, profile, pages, navbar model advice.
- `rbac`: roles, permissions and initialization.
- `item`: paginated items/comprovantes, attachments, download, observation, verification, descriptions, document types.
- `home`: dashboard and initial cards.
- `relatorio`: financial summary, Playwright PDF and executive mock.
- `notificacao`: notifications derived from receitas.
- `security`: JWT/session filter, rate limit, headers, CORS, CSRF, secret admin routes.
- `monitoring`: query count per request, HTTP timing, JVM memory, Micrometer export.
- `common`: shared utilities such as revenue classification and technical-role filtering.

## Architecture Rules

- Controllers must be in `..controller..` and use `*Controller`.
- Services must be in `..service..` and use `*Service`.
- Repositories must be in `..repository..` and use `*Repository`.
- Preferred flow: `controller -> service -> repository`.
- `ArchitectureRulesTest` and the Maven pipeline are the source of truth.
